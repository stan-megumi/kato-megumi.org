1 introduction

BattleMain 继承于 GameStateMain, 整个游戏都是同过改变某个State然后不断的 Update()来实现游戏的运行， 这些包括进入，退出，更新，退出后处理，变更状态等多个method。BattleMain 继承了这一点，并通过重写 (override) 其中的一些 method 来实现交互：

  • BattleMain 最重要的成员就是 BattleSystem
  • BattleMain 最核心的工作就是 Update, 其中最重要的一步就是 BattleSystem.SystemUpdate()
  • BattleMain.Start() 如下
    1. NextState 初始化是 -1, 并被改变为 1
    2. BattleMain.Update() 进入 GameStateMain.Update()，NextState被重置-1
    3. 由于NextState 的改变，保存了当前状态之后， 我们进入.ChangeState()
    4. 它已被改写为BattleMain.ChangeState()
    5. 初始化后进入核心的 BattleSystem.SystemUpdate()
    6. 并且由于NextState被重置-1，第3,4步被跳过， 以后的BattleMain.Update() 等价于BattleSystem.SystemUpdate()

1.2 methods

private void Start() {base.SetNextState(1);}

protected override void Update()
{

        base.Update();  // GameStateMain.Update();

        this.m_System.SystemUpdate();
}

private void LateUpdate()
{
        this.m_System.SystemLateUpdate();
}

public override GameStateBase ChangeState(int stateID)
{
        switch (base.m_NextStateID)
        {
        case 1:
                return new BattleState_Main(this); //初始化程序
        case 2:
                return new BattleState_Result(this);
        case 3:
                return new BattleState_Final(this);
        default:
                return null;
        }
}
